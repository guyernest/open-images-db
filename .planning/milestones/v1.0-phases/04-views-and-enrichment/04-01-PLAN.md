  <action>
Create `queries/views/` directory and 4 SQL files. Each file contains a single `CREATE OR REPLACE VIEW __DATABASE__.<view_name> AS` statement. Use __DATABASE__ placeholder (NOT hardcoded `open_images`). No __BUCKET__ needed -- views reference tables, not S3.

**01-labeled-images.sql** (VIEW-01): View `labeled_images`. 3-way JOIN: labels + images + class_descriptions. Columns: image_id, original_url, original_landing_url, license, author, title, original_size, thumbnail_300k_url, rotation, source, label_name, display_name, confidence. JOIN on l.image_id = i.image_id AND l.label_name = cd.label_name. Combines human AND machine labels (source column distinguishes them).

**02-labeled-boxes.sql** (VIEW-02): View `labeled_boxes`. 3-way JOIN: bounding_boxes + images + class_descriptions. All bounding_boxes columns + image metadata + display_name + computed geometry:
- `(x_max - x_min) * (y_max - y_min) AS box_area`
- `(x_max - x_min) AS box_width`
- `(y_max - y_min) AS box_height`
- `(x_min + x_max) / 2.0 AS box_center_x`
- `(y_min + y_max) / 2.0 AS box_center_y`
- `CASE WHEN (y_max - y_min) > 0 THEN (x_max - x_min) / (y_max - y_min) ELSE NULL END AS aspect_ratio`

Use `bb.` prefix for bounding_boxes columns.

**03-labeled-masks.sql** (VIEW-03 + MASK-01): View `labeled_masks`. 3-way JOIN: masks + images + class_descriptions. All masks columns + image metadata + display_name + computed geometry from box coordinates (same formulas as boxes but using m.box_x_min etc) + click_count:
- `CASE WHEN m.clicks IS NULL OR m.clicks = '' THEN 0 ELSE cardinality(split(m.clicks, ';')) END AS click_count`

IMPORTANT: clicks column is NOT JSON. It is semicolon-delimited `X Y T;X Y T;...` format. Do NOT use json_extract. Use split() and cardinality() per research findings.

**04-labeled-relationships.sql** (VIEW-04): View `labeled_relationships`. JOIN relationships + images + class_descriptions TWICE (cd1 for label_name_1, cd2 for label_name_2). Columns: image_id, image metadata, label_name_1, display_name_1, label_name_2, display_name_2, relationship_label, all coordinate columns (x_min_1, x_max_1, y_min_1, y_max_1, x_min_2, x_max_2, y_min_2, y_max_2).

Each SQL file should have a header comment with the view name, join description, and requirement ID(s). Follow the exact SQL patterns from 04-RESEARCH.md code examples.
