<task type="auto" tdd="true">
  <name>Task 2: Implement OpenImagesStack to pass all assertion tests</name>
  <files>infra/lib/open-images-stack.ts</files>
  <action>
    Implement the full stack in `infra/lib/open-images-stack.ts` following the research patterns exactly. Single file, logical sections:

    **Tags (all resources):**
    - project: open-images, environment: production, owner: data-team, cost-center: analytics

    **S3 Bucket (INFRA-01, INFRA-04):**
    - bucketName: `open-images-${this.account}-${this.region}`
    - encryption: S3_MANAGED (SSE-S3)
    - versioned: false
    - blockPublicAccess: BLOCK_ALL
    - removalPolicy: DESTROY
    - autoDeleteObjects: true
    - Note: Do NOT create prefix resources — `raw/`, `warehouse/`, `athena-results/` are virtual, created when objects are uploaded

    **Glue Database (INFRA-02):**
    - Use `glue.CfnDatabase` (L1 construct, not alpha package — research recommends L1 for plain database)
    - catalogId: this.account
    - name: 'open_images' (lowercase, underscores — NOT hyphens or mixed case per Pitfall 2)
    - locationUri: `s3://${bucket.bucketName}/warehouse/` (trailing slash per Pitfall 5)
    - description: 'Open Images V7 annotation data in Iceberg format'

    **Athena Workgroup (INFRA-03):**
    - Use `athena.CfnWorkGroup`
    - name: 'open-images'
    - state: ENABLED
    - recursiveDeleteOption: true (allows cdk destroy per INFRA-04)
    - bytesScannedCutoffPerQuery: 10 * 1024 * 1024 * 1024 (10 GB)
    - enforceWorkGroupConfiguration: true (per Pitfall 6 — without this, queries can override limits)
    - publishCloudWatchMetricsEnabled: true
    - engineVersion.selectedEngineVersion: 'Athena engine version 3' (required for Iceberg per Pitfall 6)
    - resultConfiguration.outputLocation: `s3://${bucket.bucketName}/athena-results/` (trailing slash per Pitfall 4; using athena-results/ not queries/ per research Open Question 1)
    - resultConfiguration.encryptionConfiguration.encryptionOption: SSE_S3

    **IAM Policy (INFRA-05):**
    - Create a managed policy (not a role — per research Open Question 2, Athena runs under caller's identity)
    - Policy name: 'open-images-athena-access'
    - S3 permissions: s3:GetObject, s3:PutObject, s3:ListBucket, s3:GetBucketLocation, s3:DeleteObject on the bucket and bucket/*
    - Glue permissions: glue:GetDatabase, glue:GetDatabases, glue:GetTable, glue:GetTables, glue:GetPartition, glue:GetPartitions, glue:CreateTable, glue:UpdateTable, glue:DeleteTable, glue:BatchGetPartition
    - Glue resources scoped to: catalog, database/open_images, table/open_images/*
    - Athena permissions: athena:StartQueryExecution, athena:GetQueryExecution, athena:GetQueryResults, athena:StopQueryExecution, athena:GetWorkGroup scoped to the workgroup ARN

    **Outputs:**
    - BucketName (exportName: open-images-bucket-name)
    - BucketArn (exportName: open-images-bucket-arn)
    - DatabaseName = 'open_images' (exportName: open-images-database-name)
    - WorkgroupName = 'open-images' (exportName: open-images-workgroup-name)
    - AthenaAccessPolicyArn (exportName: open-images-athena-policy-arn)

    After implementation, run tests — all MUST PASS (GREEN phase).

    Then run `cd infra && npx cdk synth --quiet` to verify template synthesis.
