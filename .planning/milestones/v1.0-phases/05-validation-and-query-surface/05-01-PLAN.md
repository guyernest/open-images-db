**Script structure:**
1. Shebang, set -euo pipefail, header comment block with Usage/Options (--quick, --help)
2. Source common.sh + athena.sh
3. Define readonly arrays for table-to-raw mappings
4. Argument parsing (--quick, --help) using the same sed pattern for --help
5. main() function with:

**Phase 1: Row count validation (VAL-01)**
- Compare COUNT(*) between each Iceberg table and its raw_* counterpart
- Table-to-raw mapping (use readonly associative-style arrays or parallel arrays):
  - images -> raw_images
  - class_descriptions -> raw_class_descriptions
  - labels -> raw_labels_human + raw_labels_machine (SUM both)
  - bounding_boxes -> raw_bounding_boxes
  - masks -> raw_masks
  - relationships -> raw_relationships
  - label_hierarchy -> raw_label_hierarchy
- For labels (special case): raw count = `SELECT (SELECT COUNT(*) FROM ${ATHENA_DATABASE}.raw_labels_human) + (SELECT COUNT(*) FROM ${ATHENA_DATABASE}.raw_labels_machine)`
- Tolerance: 1% threshold. Exact match = PASS, within 1% = WARN, above 1% = FAIL
- Use the tolerance comparison pattern from RESEARCH.md (abs_diff, threshold calc)

**Phase 2: Spot-check validation (VAL-02) -- skip in --quick mode**
- Pick a sample image_id from bounding_boxes table: `SELECT image_id FROM ${ATHENA_DATABASE}.bounding_boxes LIMIT 1`
- For that image_id, verify:
  - Coordinate columns (x_min, x_max, y_min, y_max) are BETWEEN 0.0 AND 1.0
  - Boolean columns (is_occluded, is_truncated, is_group_of, is_depiction, is_inside) are not null
- Pick a sample image_id from labels table and verify:
  - confidence is BETWEEN 0.0 AND 1.0
  - source is not null and not empty
- Pick a sample from masks table and verify:
  - predicted_iou is BETWEEN 0.0 AND 1.0
  - mask_path is not null and not empty

Use CASE WHEN ... THEN 'VALID' ELSE 'INVALID' END pattern for spot-check queries.

**Summary:** Pass/fail/warn counters, elapsed time, non-zero exit on failures.

```
# Example SQL Queries

Brief intro: These queries run against the open_images database via the open-images Athena workgroup.

## Single-Table Queries

1. **List all images with metadata** -- images table, basic SELECT with LIMIT
2. **Look up a class name by MID** -- class_descriptions table, WHERE filter
3. **Count labels by source type** -- labels table, GROUP BY source
4. **Find large bounding boxes** -- bounding_boxes table, computed area filter
5. **List masks with high IoU** -- masks table, WHERE predicted_iou filter
6. **Count visual relationships by type** -- relationships table, GROUP BY relationship_label
7. **Find child classes of a parent** -- label_hierarchy + class_descriptions join

## Cross-Table Joins

8. **Find images labeled "Dog" with bounding boxes** -- Use labeled_boxes view, WHERE display_name = 'Dog'
9. **Images with both labels and segmentation masks** -- Join labels + masks on image_id, join class_descriptions for names
10. **Visual relationships with human-readable labels** -- Use labeled_relationships view

## String Field Parsing

11. **Parse semicolon-delimited clicks column** -- Use split(clicks, ';') and cardinality() on masks or labeled_masks view
12. **Count clicks per mask** -- Aggregate click_count from labeled_masks view
```
