   - Query 1: Find all "Person on Horse" relationships using hierarchy_relationships view
     ```sql
     SELECT display_name_1, display_name_2, relationship_label,
            ancestor_name_1, ancestor_name_2, depth_1, depth_2
     FROM __DATABASE__.hierarchy_relationships
     WHERE ancestor_name_1 = 'Person'
       AND ancestor_name_2 = 'Horse'
       AND relationship_label IN ('on', 'ride', 'interacts_with')
     ORDER BY display_name_1, relationship_label;
     ```

     ```sql
     ORDER BY instance_count DESC;
     ```

2. queries/examples/02-hierarchy-browsing.sql:
   - Header: "Navigate the class hierarchy -- roots, subtrees, paths"
   - Query 1: Find root node(s)
     ```sql
     SELECT mid, display_name FROM __DATABASE__.class_hierarchy WHERE depth = 0;
     ```
   - Expected: Entity (single root)
   - Query 2: Browse children of a class (e.g., Person)
     ```sql
     SELECT display_name, depth, edge_type, is_leaf
     FROM __DATABASE__.class_hierarchy
     WHERE parent_name = 'Person'
     ORDER BY display_name;
     ```
   - Expected: Man, Woman, Girl, Boy (all subcategory, all leaves)
   - Query 3: Full path from root to a specific class
     ```sql
     SELECT display_name, depth, root_path
     FROM __DATABASE__.class_hierarchy
     WHERE display_name = 'Man';
     ```
   - Expected: root_path = "Entity > Person > Man", depth = 2
   - Query 4: All leaf nodes at maximum depth
     ```sql
     SELECT display_name, depth, root_path
     FROM __DATABASE__.class_hierarchy
     WHERE is_leaf = true
     ORDER BY depth DESC, display_name
     LIMIT 20;
     ```

3. queries/examples/03-relationship-discovery.sql:
   - Header: "Discover relationship types between parent classes"
   - Query 1: What relationships exist between Person and Animal?
     ```sql
     SELECT relationship_label, COUNT(*) AS instance_count
     FROM __DATABASE__.hierarchy_relationships
     WHERE ancestor_name_1 = 'Person'
       AND ancestor_name_2 = 'Animal'
     GROUP BY relationship_label
     ORDER BY instance_count DESC;
     ```
   - Query 2: Top entity pairs by relationship count (using ancestor classes)
     ```sql
     SELECT ancestor_name_1, ancestor_name_2, relationship_label,
            COUNT(*) AS instance_count
     FROM __DATABASE__.hierarchy_relationships
     WHERE depth_1 = 1 AND depth_2 = 1
     GROUP BY ancestor_name_1, ancestor_name_2, relationship_label
     ORDER BY instance_count DESC
     LIMIT 20;
     ```
   - Comment: Shows relationships at the "parent class" level of abstraction

4. queries/examples/04-subtree-statistics.sql:
   - Header: "Count images and relationships per hierarchy branch"
   - Query 1: Relationship count per top-level class (as subject)
     ```sql
     SELECT ancestor_name_1 AS class_name, COUNT(*) AS relationship_count
     FROM __DATABASE__.hierarchy_relationships
     WHERE depth_1 > 0
     GROUP BY ancestor_name_1
     ORDER BY relationship_count DESC
     LIMIT 15;
     ```
   - Query 2: Classes with most descendants
     ```sql
     SELECT ch.display_name, COUNT(*) AS descendant_count
     FROM __DATABASE__.class_hierarchy ch
     JOIN __DATABASE__.class_hierarchy desc
       ON desc.root_path LIKE '%' || ch.display_name || '%'
       AND desc.depth > ch.depth
     GROUP BY ch.display_name
     ORDER BY descendant_count DESC
     LIMIT 10;
     ```
   - Or simpler: count children per parent from class_hierarchy
     ```sql
     SELECT parent_name, COUNT(*) AS child_count
     FROM __DATABASE__.class_hierarchy
     WHERE parent_name IS NOT NULL
     GROUP BY parent_name
     ORDER BY child_count DESC
     LIMIT 15;
     ```
