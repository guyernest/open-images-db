  <files>scripts/create-tables.sh, scripts/lib/reorganize-raw.sh</files>
  <action>
Create `scripts/lib/reorganize-raw.sh` with a `reorganize_raw_data()` function that copies each raw CSV from its shared S3 prefix into a dedicated sub-prefix so Athena external tables can point to exactly one file per LOCATION. Use `aws s3 cp` (not mv -- keep originals for safety). The mapping is:

```
s3://BUCKET/raw/metadata/validation-images-with-rotation.csv -> s3://BUCKET/raw/tables/images/validation-images-with-rotation.csv
s3://BUCKET/raw/metadata/oidv7-class-descriptions.csv -> s3://BUCKET/raw/tables/class_descriptions/oidv7-class-descriptions.csv
s3://BUCKET/raw/annotations/oidv7-val-annotations-human-imagelabels.csv -> s3://BUCKET/raw/tables/labels_human/oidv7-val-annotations-human-imagelabels.csv
s3://BUCKET/raw/annotations/oidv7-val-annotations-machine-imagelabels.csv -> s3://BUCKET/raw/tables/labels_machine/oidv7-val-annotations-machine-imagelabels.csv
s3://BUCKET/raw/annotations/validation-annotations-bbox.csv -> s3://BUCKET/raw/tables/bounding_boxes/validation-annotations-bbox.csv
s3://BUCKET/raw/annotations/validation-annotations-object-segmentation.csv -> s3://BUCKET/raw/tables/masks/validation-annotations-object-segmentation.csv
s3://BUCKET/raw/annotations/oidv6-validation-annotations-vrd.csv -> s3://BUCKET/raw/tables/relationships/oidv6-validation-annotations-vrd.csv
```

Make it idempotent: check if destination exists before copying (use `aws s3 ls` to check). Source common.sh for logging and AWS_PROFILE.

Create `scripts/create-tables.sh` as the main runner script:

1. Source common.sh, reorganize-raw.sh, flatten-hierarchy.sh
2. Parse optional `--bucket BUCKET` argument (pass through to discover_bucket)
3. Call `check_prerequisites` (also check `jq` is available)
4. Call `discover_bucket` to get the bucket name
5. Call `reorganize_raw_data "$bucket"` to set up per-table S3 prefixes
6. Call `flatten_hierarchy "$bucket"` to download hierarchy JSON, flatten to CSV, upload to S3
7. For each SQL file in `queries/tables/` (sorted by filename):
   - Read the file content
   - Split on semicolons to get individual statements (trim whitespace, skip empty)
   - For each statement, call `run_athena_query "$statement" "$description"`
   - The `run_athena_query` function does:
     a. Substitute `__BUCKET__` placeholder in SQL with actual bucket name
     b. `aws athena start-query-execution --query-string "$sql" --work-group "open-images" --query-execution-context "Database=open_images,Catalog=AwsDataCatalog" --profile "$AWS_PROFILE" --output text --query 'QueryExecutionId'`
     c. Poll with `aws athena get-query-execution` every 2 seconds until status is not RUNNING/QUEUED
     d. If SUCCEEDED, log success and return 0
     e. If FAILED/CANCELLED, log the StateChangeReason and return 1
8. Track and report total tables created, failures. Exit non-zero if any failure.
9. Runner continues on failure (processes all SQL files, reports all failures at end) per Claude's discretion on error handling.

Important: SQL files use `__BUCKET__` as a placeholder for the S3 bucket name. The runner substitutes this at execution time using `sed "s|__BUCKET__|${bucket}|g"`.

Make the script executable (chmod +x). Follow established patterns from download-all.sh (argument parsing style, sourcing pattern, logging).

<task type="auto">
  <name>Task 2: Create hierarchy flattener and verification script</name>
  <files>scripts/lib/flatten-hierarchy.sh, scripts/verify-tables.sh</files>
  <action>
Create `scripts/lib/flatten-hierarchy.sh` with a `flatten_hierarchy()` function that:
1. Takes `$bucket` as argument
2. Downloads `https://storage.googleapis.com/openimages/2018_04/bbox_labels_600_hierarchy.json` to a temp file
3. Uses jq recursive descent to extract parent-child edges from both `Subcategory` and `Part` arrays:
   ```bash
   jq -r '
     def edges:
       .LabelName as $parent |
       ((.Subcategory // [])[] | "\($parent),\(.LabelName)", edges),
       ((.Part // [])[] | "\($parent),\(.LabelName)", edges);
     edges
   ' "$temp_file"
   ```
4. Writes output to a temp CSV file with header line `parent_mid,child_mid`
5. Uploads to `s3://$bucket/raw/tables/label_hierarchy/label_hierarchy.csv`
6. Cleans up temp files
7. Idempotent: always regenerates (fast operation)

Create `scripts/verify-tables.sh` that validates all 7 Iceberg tables exist and have correct data:
1. Source common.sh, discover bucket
2. Accept `--quick` flag (only checks first table found for fast feedback) and optional `--bucket` flag
3. Define the 7 expected tables: images, class_descriptions, labels, bounding_boxes, masks, relationships, label_hierarchy
4. For each table:
   a. Run `SELECT COUNT(*) as cnt FROM open_images.{table}` via Athena (reuse the run_athena_query pattern -- or implement a simpler version that also captures the query result)
   b. To get the result, after the query succeeds, use `aws athena get-query-results --query-execution-id "$query_id"` to read the count
   c. Verify count > 0
5. For type checking (when not --quick):
   a. Run `SELECT typeof(x_min) as t FROM open_images.bounding_boxes LIMIT 1` -- verify returns "double"
   b. Run `SELECT typeof(is_occluded) as t FROM open_images.bounding_boxes LIMIT 1` -- verify returns "boolean"
   c. Run `SELECT typeof(confidence) as t FROM open_images.labels LIMIT 1` -- verify returns "double"
6. Report pass/fail per table, exit non-zero if any check fails

Make both scripts executable. The verify script needs its own `run_athena_query_with_result()` function that returns the query result, not just success/failure. This can be a helper in the verify script itself or added to common.sh.
