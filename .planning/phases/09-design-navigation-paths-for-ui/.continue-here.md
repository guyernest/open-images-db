<decisions_made>

- MCP prompts are server-orchestrated (not LLM-decided) — fundamental protocol distinction
- find_images uses structured args (subject/object/relationship) so LLM does NL parsing, server does symbolic computation
- resolve_entity is a lightweight lookup tool for when the LLM is unsure of canonical class names
- popular_entities resource lets LLM pre-cache common names at conversation start
- Server is stateless — no session, no previous_query concept
- narrow_results should be removed — just use find_images for everything (widget holds state)
- subject/object/relationship should accept arrays for multi-select facets (OR within dimension, AND across)

</decisions_made>
