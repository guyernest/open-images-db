**A. `design/navigation-flows/01-search-flow.md`** -- Initial search entry

Structure as a step-by-step flow with branching paths. For each step show:
- **Actor**: User / LLM / MCP Server / Widget
- **Action**: What they do
- **Data**: What data is passed or produced
- **Next**: What happens next (may branch)

**Flow 1A: MCP Prompt Entry** (e.g., user types `/find_images dogs`)
1. User: Types `/find_images dogs` in conversation
2. LLM: Recognizes MCP prompt, calls find_images tool with query="dogs"
3. MCP Server: Resolves "dogs" to class hierarchy (Dog under Entity > Animal > Carnivore), queries labeled_images + class_hierarchy + labeled_relationships views, assembles three-layer response
4. Widget (results-grid): Renders in conversation showing:
   - Top bar: "523 dog images found"
   - Facet pills: Poodle, German Shepherd, Labrador (from hierarchy children), ride, on, wears (from relationships)
   - 3x grid of thumbnails with label overlays
   - "Load more" at bottom
5. LLM conversation text: "Found 523 images of dogs across 12 breeds. The most common are Poodle (89), German Shepherd (67), and Labrador (54). You can click a breed to narrow results, or tell me what you're looking for."

**Flow 1B: Natural Language Entry** (e.g., "I need photos of people riding horses")
1. User: Types "I need photos of people riding horses"
2. LLM: Interprets intent, calls find_images with query="people riding horses" (or uses relationship filter: relationship="ride")
3. MCP Server: Uses hierarchy_relationships view (ancestor_name_1='Person', ancestor_name_2='Horse', relationship_label IN ('ride', 'on')), maps query pattern to example 01-people-on-horses.sql
4. Widget: Same results-grid but facets show Person subcategories (Man, Woman, Boy, Girl) and relationship types (ride, on, interacts_with)
5. LLM text: More conversational response based on results

**Flow 1C: Broad/Ambiguous Query** (e.g., "show me animals")
1. User: Types "show me animals"
2. LLM: Recognizes broad query. Two strategies:
   - Strategy A: Call find_images with query="Animal", return top results + suggest narrowing
   - Strategy B: Call explore_category with class_name="Animal" to show the hierarchy first
3. LLM text: "I found over 2000 animal images. The Animal category includes birds, carnivores, mammals, and more. Would you like to browse by category, or shall I show you results? You can also ask for specific animals like 'dogs' or 'eagles'."

**Example query mapping:** Note which example queries (01-08) map to this flow:
- 01-people-on-horses.sql -> Flow 1B (relationship query)
- 05-entity-search.sql -> Flow 1A (direct class search)
- 08-relationship-inventory.sql -> Flow 1B (relationship-based search)

**B. `design/navigation-flows/02-refinement-flow.md`** -- Narrowing results

**Flow 2A: Facet Click Refinement** (within widget, no new conversation turn)
1. User: Clicks "Poodle" facet pill in results-grid widget
2. Widget: Calls tools/call narrow_results with filter="category:Poodle", previous_query="dogs"
3. MCP Server: Adds WHERE clause for Poodle class, re-queries
4. Widget: Re-renders in place. Grid shows only poodle images. "Poodle" pill is now active (filled). Count updates to "89 poodle images". Other facets update to reflect poodle-specific relationships.
5. No new conversation turn. LLM is NOT involved.

**Flow 2B: Free-Text Follow-Up** (new conversation turn)
1. User: Types "I see poodles and german shepherds, show me more poodles" in conversation
2. LLM: Reads conversation context (previous find_images results visible), calls find_images or narrow_results with appropriate filter
3. MCP Server: Queries for poodle-specific images
4. Widget: NEW results-grid widget appears below in conversation. Previous widget remains frozen.
5. LLM text: "Here are 89 poodle images. I can also show you poodles in specific situations -- for example, poodles with people, or outdoor scenes."

**Flow 2C: Multi-Facet Refinement**
1. User: Clicks "Poodle" facet, then clicks "wears" relationship facet
2. Widget: Calls narrow_results with filter="category:Poodle,relationship:wears"
3. Widget: Shows poodles in "wears" relationships. Both pills active.

**Flow 2D: Removing Filters**
1. User: Clicks active "Poodle" pill (toggle off)
2. Widget: Calls narrow_results removing that filter
3. Widget: Returns to broader result set

**Flow 2E: Pagination**
1. User: Clicks "Load more"
2. Widget: Calls narrow_results with page=2, same filters
3. Widget: Appends new images below existing ones

**Example query mapping:**
- 03-relationship-discovery.sql -> Flow 2A (relationship facet)
- 06-category-exploration.sql -> Flow 2A (category facet)

**Key design note:** Emphasize that Flow 2A (facet click) is the PRIMARY refinement mechanism because it uses tools/call (reliable, fast, no LLM involvement). Flow 2B (free-text) goes through the LLM and produces a new widget, which is fine but slower and less predictable.

Create two conversation flow documents:

**A. `design/navigation-flows/03-detail-flow.md`** -- Image detail view and navigate-from-image

**Flow 3A: Entering Detail View** (from results grid)
1. User: Clicks a thumbnail in the results-grid widget
2. Widget: Sends ui/message: "Show details for image {image_id}"
3. LLM: Calls get_image_details with image_id
4. MCP Server: Queries labeled_images, labeled_boxes, labeled_relationships, labeled_masks for that image_id (maps to pattern from 07-image-contents.sql). Assembles full annotation data.
5. Widget (image-detail): Renders below in conversation. Shows:
   - Full-size image with bounding box overlays
   - Layer toggles (boxes, labels, relationships)
   - Metadata panel with labels table, relationships list, image info
   - Navigate-from-image action buttons
6. LLM text: "This image contains a Man riding a Horse outdoors. The Man is wearing a Helmet. There are 3 labeled objects, 2 relationships, and 4 bounding boxes. You can click on any label to find more images of that type."

**Flow 3B: Navigate-From-Image -- "More with [label]"**
1. User: Clicks "More with Dog" button in image-detail widget
2. Widget: Sends ui/message: "Find more images of Dog"
3. LLM: Calls find_images with query="Dog"
4. New results-grid widget appears below with dog images
5. Fallback: Widget shows "Try typing: find more images of Dog" after 5s

**Flow 3C: Navigate-From-Image -- "Similar scenes"**
1. User: Clicks "Similar scenes" button
2. Widget: Sends ui/message with the image's label set: "Find images containing Dog, Person, and Outdoors"
3. LLM: Calls find_images with multi-class query
4. New results-grid appears with images sharing those labels

**Flow 3D: Navigate-From-Image -- "Explore [category]"**
1. User: Clicks "Explore Animal" button (derived from hierarchy path of the Dog label)
2. Widget: Sends ui/message: "Explore the Animal category"
3. LLM: Calls explore_category with class_name="Animal"
4. New hierarchy-browser widget appears

**Flow 3E: Interacting with Annotations**
1. User: Clicks on a bounding box label in the overlay
2. Widget: Sends ui/message: "Find more images of {display_name}"
3. Alternatively, user clicks on a relationship in the info panel
4. Widget: Sends ui/message: "Find images where {subject} {relationship} {object}"

**Flow 3F: ui/message Fallback**
All navigate-from-image actions use ui/message. Document the fallback behavior:
1. Widget sends ui/message
2. After 5 seconds with no new conversation turn: widget shows "It looks like that didn't work. Try typing: {the exact text that was sent}"
3. This text is copyable (click to copy to clipboard)
4. User pastes and sends -> LLM processes normally

**Example query mapping:**
- 07-image-contents.sql -> Flow 3A (multi-table image join)
- 01-people-on-horses.sql -> Flow 3C (relationship-based similar scenes)

**B. `design/navigation-flows/04-hierarchy-flow.md`** -- Category browsing and code mode

**Flow 4A: Direct Hierarchy Entry** (user asks about categories)
1. User: Types "What kinds of animals are in the dataset?" or `/explore_category Animal`
2. LLM: Calls explore_category with class_name="Animal", depth=2
3. MCP Server: Queries class_hierarchy for Animal subtree (maps to 02-hierarchy-browsing.sql), gets sample images for each subcategory, gets relationship summary (maps to 04-subtree-statistics.sql)
4. Widget (hierarchy-browser): Renders showing:
   - Breadcrumb: Entity > Animal
   - Node header: "Animal -- 2,847 images, 15 relationship types"
   - Children: Bird (expand arrow, 3 thumbnails), Carnivore (expand arrow, 3 thumbnails), Mammal, Invertebrate, Reptile, Fish
   - "View all animal images" button
5. LLM text: "The Animal category has 6 main branches: Bird, Carnivore, Mammal, Invertebrate, Reptile, and Fish. Carnivore is the largest with 1,200+ images including dogs, cats, bears, and more. Click a category to explore deeper, or click 'View all images' to see them all."

**Flow 4B: Drilling Down the Hierarchy**
1. User: Clicks "Carnivore" in the tree
2. Widget: Calls tools/call explore_category with class_name="Carnivore", depth=2
3. Widget: Re-renders in place. Breadcrumb updates to Entity > Animal > Carnivore. Shows Dog, Cat, Bear, Fox, Lion, Tiger children with sample thumbnails.
4. No new conversation turn.
5. User: Clicks "Dog"
6. Widget: Calls tools/call explore_category for Dog. Shows Poodle, German Shepherd, Labrador, etc. (leaf nodes with no expand arrows).

**Flow 4C: Transitioning from Hierarchy to Image Grid**
1. User: Clicks "View all images" on the Dog category
2. Widget: Calls tools/call find_images with category filter "Dog"
3. Widget: Internal mode transition -- tree view fades, grid view appears within same iframe
4. Grid shows dog images with subcategory facets (Poodle, German Shepherd, etc.)
5. User: Clicks "Back to tree" button
6. Widget: Returns to tree mode with previous state intact

**Flow 4D: Navigating Up via Breadcrumb**
1. User: Clicks "Animal" in breadcrumb trail (while viewing Dog subtree)
2. Widget: Calls tools/call explore_category for Animal
3. Widget: Re-renders at Animal level

**Flow 4E: Code Mode Entry**
1. User: Types `/start_code_mode`
2. LLM: MCP prompt triggers. 00-mcp-reference.sql is loaded as context resource. LLM responds with code mode instructions.
3. LLM text: "Code mode active. I have the full Open Images schema loaded. You can ask me to write SQL queries for anything -- relationships, hierarchy traversal, image analysis, statistics. Try: 'How many images show a person riding a horse?' or 'What are the most common object combinations?'"
4. User: "How many images show a person riding a horse?"
5. LLM: Generates SQL using hierarchy_relationships view, ancestor expansion pattern from 00-mcp-reference.sql Section 3. Returns SQL + results in conversation text (no widget for code mode).
6. Code mode response includes: the SQL query (in code block), the explanation of what it does, the results, and suggestions for follow-up queries.

**Flow 4F: Code Mode to Tool Mode Transition**
1. User (in code mode): "Actually, can you show me those horse riding images?"
2. LLM: Recognizes intent to switch to visual mode. Calls find_images with appropriate query.
3. Results-grid widget appears. User is effectively back in tool mode.
4. LLM text: "Here are the images. You're now in visual browsing mode -- click any image for details, or use the facets to filter."
