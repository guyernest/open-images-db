  <files>scripts/lib/common.sh, scripts/lib/download-annotations.sh, scripts/lib/download-metadata.sh, scripts/lib/download-masks.sh</files>
  <action>
Create the scripts/lib/ directory and four shell library files. All files should use `#!/usr/bin/env bash` and `set -euo pipefail`.

**scripts/lib/common.sh** -- Shared functions:
- `log_info`, `log_error`, `log_warn` -- Prefixed logging to stderr
- `check_prerequisites` -- Verify curl, aws, jq, unzip are installed; verify AWS credentials work with profile `ze-kasher-dev`; fail with install instructions if missing (DATA-06)
- `discover_bucket` -- Get bucket name from CloudFormation stack outputs (stack: OpenImagesStack, output key: BucketName, profile: ze-kasher-dev). Accept `--bucket` override parameter for flexibility. Fail with clear error if stack not found.
- `download_file` -- Download a single file from URL to local path using `curl -fSL --retry 3 --retry-delay 5 -z "$dest" -o "$dest" "$url"`. The `-z` flag provides idempotency (DATA-04). Verify file is non-empty after download.
- `upload_to_s3` -- Upload a local directory to an S3 prefix using `aws s3 sync --profile ze-kasher-dev`. Sync provides idempotency (DATA-04).
- `TEMP_DIR` default: `${OPEN_IMAGES_TEMP:-$HOME/open-images-tmp}` (configurable via env var to handle disk space concerns on small EC2 instances)

**scripts/lib/download-annotations.sh** -- Function `download_annotations`:
- Sources common.sh if not already loaded
- Downloads these 5 annotation CSVs to `$TEMP_DIR/annotations/`:
  1. `https://storage.googleapis.com/openimages/v7/oidv7-val-annotations-human-imagelabels.csv`
  2. `https://storage.googleapis.com/openimages/v7/oidv7-val-annotations-machine-imagelabels.csv`
  3. `https://storage.googleapis.com/openimages/v5/validation-annotations-bbox.csv`
  4. `https://storage.googleapis.com/openimages/v5/validation-annotations-object-segmentation.csv`
  5. `https://storage.googleapis.com/openimages/v6/oidv6-validation-annotations-vrd.csv`
- Shows progress: "Downloading annotations... N/5 files"
- Uploads to `s3://$BUCKET/raw/annotations/` via upload_to_s3
- Preserves original filenames (per user decision)

**scripts/lib/download-metadata.sh** -- Function `download_metadata`:
- Downloads these 3 files to `$TEMP_DIR/metadata/`:
  1. `https://storage.googleapis.com/openimages/v7/oidv7-class-descriptions.csv`
  2. `https://storage.googleapis.com/openimages/v7/oidv7-class-descriptions-boxable.csv`
  3. `https://storage.googleapis.com/openimages/2018_04/validation/validation-images-with-rotation.csv`
- Shows progress: "Downloading metadata... N/3 files"
- Uploads to `s3://$BUCKET/raw/metadata/` via upload_to_s3

**scripts/lib/download-masks.sh** -- Function `download_masks`:
- Downloads 16 zip archives from `https://storage.googleapis.com/openimages/v5/validation-masks/validation-masks-{CHAR}.zip` where CHAR is 0-9, a-f (lowercase first; if 404, try uppercase)
- Downloads zips to `$TEMP_DIR/mask-zips/`
- Extracts flat using `unzip -o -j "$zip" -d "$TEMP_DIR/masks/"` (junk paths to get flat PNGs per user decision)
- Shows progress: "Downloading mask archives... N/16"
- After all 16 extracted, uploads to `s3://$BUCKET/raw/masks/` via upload_to_s3
- Cleans up zip files after successful extraction to save disk space

All download functions use direct HTTPS URLs (per research: public bucket, no GCS auth needed). The auto-detect requester-pays decision from CONTEXT.md is satisfied because these URLs work without authentication -- if they ever fail with 403, the error message should suggest checking if the bucket became requester-pays and to install gsutil (DATA-05).

  <files>scripts/download-all.sh, scripts/README.md</files>

**scripts/download-all.sh** -- Main orchestrator script (DATA-06):
- Shebang: `#!/usr/bin/env bash`, `set -euo pipefail`
- Parse arguments: `--bucket BUCKET_NAME` (optional override), `--validate-only` (skip download, run validation only), `--skip-masks` (download annotations/metadata only, for quick testing), `--help`
- Source all lib/*.sh files
- Execution flow:
  1. `check_prerequisites` (from common.sh)
  2. `discover_bucket` (or use --bucket override)
  3. Create temp directories: `$TEMP_DIR/{annotations,metadata,masks,mask-zips}`
  4. `download_annotations "$TEMP_DIR" "$BUCKET"`
  5. `download_metadata "$TEMP_DIR" "$BUCKET"`
  6. `download_masks "$TEMP_DIR" "$BUCKET"` (unless --skip-masks)
  7. Post-download validation (built into script):
     - Count annotation CSVs in S3 (expect 5)
     - Count metadata files in S3 (expect 3)
     - Count mask PNGs in S3 (expect ~24,730, warn if < 20,000)
     - Report total sizes per prefix
  8. Generate manifest: Use `aws s3api list-objects-v2 --bucket "$BUCKET" --prefix "raw/" --profile ze-kasher-dev` piped through jq to create manifest.json with Key, Size, LastModified fields. Upload to `s3://$BUCKET/raw/manifest.json`
  9. Print summary: total files per type, total size, elapsed time
- Make executable: `chmod +x scripts/download-all.sh`

**scripts/README.md** -- Execution instructions (DATA-06):

- What gets created: S3 layout diagram showing raw/annotations/, raw/metadata/, raw/masks/, raw/manifest.json
- Idempotency: explain that re-running is safe (curl -z skips unchanged files, aws s3 sync skips matching files)
