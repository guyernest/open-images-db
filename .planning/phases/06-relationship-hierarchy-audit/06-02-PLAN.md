**Step 2: Write report**
Create reports/06-audit-report.md with these sections:

```
# Open Images Validation Set: Relationship & Hierarchy Audit

## Executive Summary
- Total relationship rows (raw table vs view)
- Total hierarchy edges
- Key finding: whether "people on horses" exists and why/why not
- Number of dropped rows and primary cause

## 1. Relationship Types Inventory (AUDIT-01)
- Table: all distinct relationship_label values with counts (from raw table)
- Table: same from labeled_relationships view
- Delta analysis: which types lose rows through the view's INNER JOIN
- Total distinct types

## 2. Hierarchy Structure (AUDIT-02)
- Root node(s) with display names
- Max depth
- Text tree visualization (top 2-3 levels showing branching)
- Branch density: richest branches (most children) and sparsest
- Coverage gap: hierarchy covers ~600 MIDs vs ~20K in class_descriptions
- Note whether Subcategory vs Part distinction is preserved (it is NOT in the flattened CSV -- both edge types merged)

## 3. Entity-Class Pair Analysis (AUDIT-03)
- Top 20 entity pairs by instance count (with display names and relationship type)
- Person-Horse specific analysis (the motivating example)
- Distribution: how many pairs have >10 instances, >100, etc.

## 4. Dropped Row Analysis
- Count and percentage of rows lost via INNER JOIN
- Which MIDs have no class_descriptions match
- Which relationship types are most affected
- Classification: missing from source data vs lost in pipeline

## 5. Gap Classification
For each identified gap, classify as:
- **Source gap**: Data not in validation set (would appear in train/test)
- **Pipeline gap**: Data lost during processing (fixable)
- **Query gap**: Data exists but not discoverable through views (fixable in Phase 7)

## 6. Recommendations for Phase 7
- Specific fixes suggested based on findings
- Priority ordering
```
